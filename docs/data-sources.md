# 数据来源说明

## 当前数据源

当前项目统一使用 `Twelve Data API`。

原因：

- 个人项目接入成本低
- 免费额度足够当前场景
- 接口结构清晰，便于 AI 和人类共同维护
- 适合盘后日线场景

## 当前调用方式

接口：

- `time_series`

参数策略：

- `symbol=SPY` 或 `QQQ`
- `interval=1day`
- 初始化时按可行范围回补长期历史
- 日常同步只拉最近一小段增量区间

## 为什么使用 ETF 而不是指数本体

当前不是做机构级指数授权系统，而是做个人使用面板。

因此优先级是：

1. 稳定拿到可用数据
2. 低授权复杂度
3. 能支撑本地计算日 / 周 / 月涨跌

在这个前提下：

- `SPY` 足以代表标普 500
- `QQQ` 足以代表纳指 100

## 免费额度评估

当前日常同步只需要极少量请求：

- `SPY`
- `QQQ`

初始化补历史时，请求会更多一些，但总体仍然处于可控范围。当前项目的同步频率和标的数量，远低于免费额度会带来的工程压力。

## 未来切换空间

如果后续确实需要切回指数本体，可以按这个顺序升级：

1. 保留当前页面和本地计算逻辑
2. 更换同步脚本的数据来源
3. 视情况调整 symbol 与市场映射

这样可以把数据源切换控制在较小范围内，而不是重做整个项目。
